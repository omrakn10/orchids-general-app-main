3) Uygulamada kullanıcı kaydı ve giriş ekle

Senin Orchids çıktın büyük ihtimalle Vite/browser client kullanıyor. O yüzden Next.js SSR kurulumuna değil, normal browser client akışına gitmek daha doğru. @supabase/ssr özellikle SSR için cookie tabanlı istemci kurulumunda kullanılır.

src/supabase.js şöyle kalsın:

import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)

.env.local yerine Vite kullanıyorsan çoğu projede VITE_... değişkenleri gerekir. Senin Orchids çıktısında da zaten browser client oluşturulduğu söylenmiş. createClient(url, key) kullanımı Supabase JS’in standart istemci kurulumudur.

4) Signup / login fonksiyonları

Basit email-password akışı ekleyebilirsin:

import { supabase } from './supabase'

export async function signUp(email, password) {
  const { data, error } = await supabase.auth.signUp({
    email,
    password,
  })
  return { data, error }
}

export async function signIn(email, password) {
  const { data, error } = await supabase.auth.signInWithPassword({
    email,
    password,
  })
  return { data, error }
}

export async function signOut() {
  const { error } = await supabase.auth.signOut()
  return { error }
}

export async function getCurrentUser() {
  const { data, error } = await supabase.auth.getUser()
  return { user: data?.user ?? null, error }
}

Supabase Auth, kullanıcı oturumu üzerinden JWT üretir; RLS policy’leri de bu kimliği kullanarak satır bazlı yetkilendirme yapar.

5) Todo eklerken user_id gönder

Görev eklerken giriş yapan kullanıcının id’sini yazman lazım:

import { supabase } from './supabase'

export async function addTodo(text, category, priority) {
  const {
    data: { user },
    error: userError,
  } = await supabase.auth.getUser()

  if (userError || !user) {
    throw new Error('Kullanıcı oturumu bulunamadı.')
  }

  const { data, error } = await supabase
    .from('todos')
    .insert([
      {
        user_id: user.id,
        text,
        category,
        priority,
        completed: false,
      },
    ])
    .select()

  return { data, error }
}

auth.users içindeki kullanıcı id’si veri satırına bağlandığında, RLS policy doğrudan o kullanıcıya göre çalışır.

6) Todo çekerken ayrıca filtre yazmasan da RLS korur

Şu sorgu yeterli olabilir:

export async function fetchTodos() {
  const { data, error } = await supabase
    .from('todos')
    .select('*')
    .order('created_at', { ascending: false })

  return { data, error }
}

Burada kod tarafında eq('user_id', user.id) yazman faydalı olabilir ama güvenlik açısından asıl koruma RLS policy’dir. Supabase dokümanlarında da veri erişiminin asıl kontrol katmanı policy’lerdir.

İstersen daha açık da yazabilirsin:

export async function fetchTodos() {
  const {
    data: { user },
  } = await supabase.auth.getUser()

  if (!user) return { data: [], error: null }

  const { data, error } = await supabase
    .from('todos')
    .select('*')
    .eq('user_id', user.id)
    .order('created_at', { ascending: false })

  return { data, error }
}
7) Oturum dinleme

Uygulama açıldığında kullanıcı giriş yaptı mı diye bakman gerekir:

import { useEffect, useState } from 'react'
import { supabase } from './supabase'

function App() {
  const [session, setSession] = useState(null)

  useEffect(() => {
    supabase.auth.getSession().then(({ data }) => {
      setSession(data.session)
    })

    const {
      data: { subscription },
    } = supabase.auth.onAuthStateChange((_event, session) => {
      setSession(session)
    })

    return () => subscription.unsubscribe()
  }, [])

  if (!session) {
    return <div>Giriş yapmanız gerekiyor.</div>
  }

  return <div>Todo uygulaması burada</div>
}

SSR tarafında session cookie ile taşınır; client-side uygulamalarda ise oturum istemci üzerinden yönetilir. Supabase’in SSR dokümanları bu ayrımı özellikle anlatıyor.

9) Güvenlikte çok önemli nokta

Browser tarafında yalnızca anon/publishable key kullanılmalı. service_role anahtarını asla frontend’e koymamalısın; Supabase dokümanları bunun yalnızca server tarafında kullanılmasını söylüyor.
